Этот пример показывает проблему race condition в многопоточности.

У нас есть общий счетчик counter. Два потока увеличивают его по 1000 раз.
Мы ожидаем результат 2000.

В первой версии программы используется операция counter++, которая не является
атомарной. Она состоит из трех шагов:
1. чтение значения из памяти
2. увеличение значения
3. запись обратно в память

Если два потока выполняют эти шаги одновременно, один из них может перезаписать
результат другого. В итоге итоговое значение counter иногда получается меньше 2000.
Это называется race condition (состояние гонки).

Во второй версии используется метод increment() с ключевым словом synchronized.
Это гарантирует, что только один поток может выполнять этот метод в один момент
времени. Остальные потоки ждут, пока метод освободится.

Благодаря этому операции с переменной counter выполняются по очереди, и итоговый
результат всегда будет корректным (2000).
